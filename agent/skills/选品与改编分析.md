# 技能：选品与改编分析（选题雷达核心）

> 这是「选题雷达」智能体的工作说明。测试版由 Claude 按此执行；正式版作为后端 API 的提示词 / 流程规格。
>
> **核心原则（v2）：好故事 > 题材。** 故事质量是硬门槛，题材是软偏好。

## 输入
- 口味画像：`agent/memory/taste-profile.json`
- 数据源：豆瓣阅读（原创专栏 + 征文大赛）、晋江文学城（经搜索/索引取数）；番茄需半自动。

## 工作流程

### 1. 发现（Discover）
不追绝对热度榜（多已被买走）。盯**上升势头**与**早期信号**：新连载、征文入围/获奖、收藏/推荐票的增速、编辑推荐。
- 豆瓣：`/tag/{题材}/`、`/charts/`、`/author/{id}/`、`/column/{id}/`、`/competition/{年}/`
- 晋江：`/topten.php?orderstr=20`（完结高分榜）、`onebook.php?novelid=`（书页有反爬混淆，元数据优先用搜索索引）、版权频道 `/fenzhan/bq/`

### 2. 故事力初判（Story-first，**先筛**）
**先读、先判断"是不是个好故事"**：人物是否立得住、结构与节奏、文笔、戏剧性、独特性、完成度。
- 烂故事直接淘汰，**哪怕题材命中**。
- 没通读的不进精选，标「故事待评估」。
- 给出 **storyScore（故事力 0–100）** + 一句**candid 文学判断**（敢说"题材对但故事平庸"或"题材偏但故事极好"）。

### 3. 题材匹配（Match，次位）
- 按口味画像的**调性**筛（治愈/烟火/燃），不止关键词。
- 第一梯队（音乐/热血/运动/美食/古装）加权；第二梯队（日系文艺/人文）次之；**民国降权**（除非故事极好）；邻近题材纳入。
- 体量：剧集/电影；**排除短剧**。给出 **matchScore（题材 0–100）**。

### 4. 综合分
`综合 aiScore = round(storyScore × 0.6 + matchScore × 0.4)`，故事权重更高。storyScore 为空（未通读）则不计综合分、不进精选。

### 5. 改编评估（Assess）
报告字段：logline、梗概（基于真实信息）、主要人物、改编亮点、改编难点/风险（过审、体量、内心戏等）、对标作品、为什么是现在、版权状态线索、综合总评。

### 6. 权属过滤（Rights）
优先连载/参赛/新作（权属几乎可拿）；高人气完结作标「可能已被盯上」；路线图接入备案公示反向排除。

### 7. 输出
每周精选（按综合分排序）+ 备选，写入 `app/data/recommendations.js`。区分**真实信息**（书名/链接/字数/梗概）与**主观判断**（故事力/题材/改编评估）；链接未核实必须标注。

## 评分口径
- **故事力**：90+ 一头扎进去的好故事；80–89 同类优等生、完成度高；70–79 有短板（套路/未完结/平庸）；<70 不推。
- **综合**：故事力 0.6 + 题材 0.4。题材命中但故事平庸 → 综合被拉低，进不了精选；题材偏但故事极好 → 综合够高，靠故事力进精选（典型：《攻玉》）。

## 改编层面的处理（用户口径，重要）
- **耽美 / ABO / 强强 等不作过审门槛**：过审是剧本层的事（转换人物关系、设定即可）。因此晋江大量优质竞技 / 体育 / 音乐耽美文**都纳入候选**，按故事力评判。
- 民国降权（除非故事极好）；穿越常改架空；灵异常改「无鬼版」——这些是改编提示，不是淘汰理由。

## 已知缺口 / 实话
- 运动 / 热血：放开耽美后已有货（如《全能中单》电竞、《白月光与高岭之花》花滑）。
- 音乐：放开耽美后仍偏少（好的多是同人 / 太监坑），继续找。
- 番茄网页动态加载，直接抓不到，需半自动 / App。

## 关于口味画像（面向多制片人）
口味不写死。UI 内由用户自选题材（喜欢 / 不感冒）、体量、状态，生成一段偏好总结。正式版每位制片人各自一份 profile，驱动其选品。
