# 帽制片 · 需求文档（v1）

> 面向影视行业与制片人的效率工作台。本文件记录已对齐的需求，随沟通持续更新。

## 一、产品总览

分三大板块，**电脑端先做，成熟后上手机端**：

| 板块 | 目标 | 优先级 |
|------|------|--------|
| **策划** | 选书荐书：在大买家之前，发现尚未售出、有改编潜力的早期 IP | ① 先做 |
| **制作** | 剧本拆解：顺场表、人物戏量、场景 / 道具清单，进阶到并场排期 / 演员周期 | ② 往后放 |
| **发行** | 数据监测、档期竞品、宣发节点、报备与分账（待定） | ③ 最后 |

**核心定位（策划）**：不做「热度榜」——热门 IP 早被头部公司 / 平台买空。价值在「选品前置」：看上升势头而非绝对热度，靠 AI 通读判断改编潜力，替代「人气」信号。

**形态定位**：这套软件本质是一个**智能体（Agent）**，有自己的技能（skills）与记忆（memory）。
- **测试版**：由 Claude 充当大模型 / 智能体角色，人工驱动选品与分析。
- **正式版**：做 API 接口，由后端自动执行。

## 二、v1 范围：策划 · 选题雷达

### 数据源
- **豆瓣阅读**（原创专栏 + 征文大赛）+ **晋江文学城**（v2 扩源）。
- 番茄想加，但网页动态加载、直接抓不到，需半自动 / App（暂缓）。
- 后续：文学期刊中短篇。

### 选品原则（v2，最重要）
**好故事 > 题材。** 故事质量是硬门槛，题材是软偏好。每部给两个分：
- **故事力**（智能体的文学判断）+ **题材匹配**；**综合 = 故事力 × 0.6 + 题材 × 0.4**。
- 敢于「题材命中但故事平庸 → 降权不推」「题材偏但故事极好 → 加权进精选」。未通读的不进精选。

### 口味画像（选品基准）
- **第一梯队**：音乐 / 热血 / 运动 / 美食 / **古装**
- **第二梯队**：日系文艺 / 人文
- **不太感冒**：民国（除非故事极好）
- **调性暗线**：整体偏「日系青春 / 治愈 / 燃」。**按调性匹配优先于关键词**。
- **体量**：剧集或电影，**不做短剧**。
- **完结状态**：优先已完结，未完结也纳入。

### 产品形态
- **应用内浏览**：每周自动上新精选，打开即看。
- 支持：历史回看、收藏、按题材 / 来源 / 状态筛选、搜索。
- **口味偏好由用户自选**：题材（喜欢 / 不感冒）、体量、状态 → 生成偏好总结。**面向多制片人，不写死**；正式版每人一份 profile 驱动选品。
- **反馈闭环**：点赞 / 拍掉 → 持续校准（测试版存浏览器，未来回流到智能体 memory）。

### 分析报告模板（每部，已确认）
基础信息（书名 / 作者 / 链接 / 完结状态 + 字数 / 体量建议 / 题材标签 + 调性 / 时间）、一句话故事、故事梗概、主要人物、题材匹配度、改编亮点、改编难点与风险、对标作品、为什么是现在、版权状态线索、AI 综合推荐指数 + 一句话总评。

## 三、数据可行性结论

- ✅ **豆瓣阅读**：公开页面无登录墙，可拿书名 / 作者 / 链接 / 字数 / 状态 / 梗概。
- ✅ **晋江**：列表 / 排行 / 书页可访问，但**书页有反爬文字混淆**（直接抓书名会乱码）→ 元数据改用搜索索引更可靠。
- ⚠️ **番茄**：网页动态加载，直接抓为空，需半自动 / App（暂缓）。
- **音乐 / 运动 / 热血**：**耽美不作过审门槛**（用户口径：剧本层转换人物关系即可），放开后运动 / 热血已有货（晋江竞技 / 花滑耽美，如《全能中单》《白月光与高岭之花》）；音乐仍偏少。美食 + 治愈 + 古装 + 人文资源好。
- ⚠️ **版权**是全链路最弱环节，只能降低「推了才发现已售」的概率，做不到 100% 准。
- 📌 反爬加强时退一步做「半自动」：抓元数据 + 人工读正文 + AI 分析。
